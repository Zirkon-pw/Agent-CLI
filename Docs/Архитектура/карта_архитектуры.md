# Карта архитектуры

Этот раздел фиксирует целевую инженерную модель Agent CLI как приложения на Go.

Он отвечает не на вопрос "что умеет продукт", а на вопрос "как именно это должно быть собрано в коде и в файловой среде исполнения".

## Что считается ядром архитектуры

Для этого CLI архитектурным ядром являются:

- `task` как основной объект предметной области;
- встроенные и пользовательские шаблоны промптов;
- `.yml`-уточнения как отдельные прикрепляемые артефакты;
- `run` как единица исполнения;
- файловое хранилище `.agentctl/` как источник истины;
- runtime-команды `ps`, `inspect`, `watch`, `stop`, `kill`, `resume`.

## Граница между `Устройство` и `Архитектура`

Раздел `Устройство` описывает продуктовую модель:

- сущности;
- сценарии пользователя;
- формат задач;
- конфигурации;
- шаблоны промптов.

Раздел `Архитектура` переводит это в реализацию на Go:

- какие пакеты нужны;
- какие интерфейсы должны существовать;
- где должна жить state machine;
- как формируется execution contract;
- как хранятся runtime-файлы и сигналы управления.

## Главный архитектурный тезис

`agentctl` не должен быть тонкой оболочкой над LLM API.

Он должен быть локальным control plane для задач, где:

- постановка фиксируется в `task.yml`;
- поведение агента задается шаблонами;
- неясности выводятся в `.yml`-уточнения;
- выполнение наблюдаемо как runtime-процесс;
- весь жизненный цикл воспроизводим через файлы.
